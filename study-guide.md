# Gemini Web 项目学习指南 (Study Guide)

本指南旨在帮助你快速理解并掌握 `baoyu-danger-gemini-web` 项目的核心实现。该项目是一个通过逆向工程手段、在终端调用 Gemini Web 端能力的优秀案例。

---

## 1. 项目概况与核心目标

*   **项目定位**：一个基于 TypeScript 的 CLI 工具，模拟浏览器行为调用 Gemini Web API。
*   **核心优势**：无需官方 API Key，支持图片生成、视觉识别和多轮对话。
*   **关键技术**：
    *   **Cookie 绕过**：通过读取本地浏览器数据进行身份验证。
    *   **报文模拟**：逆向 Google 的特定数组格式（类似 Protobuf 的序列化格式）。
    *   **异步流处理**：处理图片生成的二进制数据。

---

## 2. 代码导航 (源码地图)

学习顺序建议：`入口` -> `业务层` -> `协议层`。

1.  **命令行入口**：`scripts/main.ts`
    *   **看点**：如何解析参数、如何处理用户同意协议（Consent Checklist）、如何调用底层 Client。
2.  **核心客户端**：`scripts/gemini-webapi/client.ts`
    *   **看点**：最重要的文件。包含了发送请求的 `ask` 方法，以及如何处理多轮对话的 Context 状态。
3.  **配置中心**：`scripts/gemini-webapi/constants.ts`
    *   **看点**：了解 Gemini 各种模型的标识符（如 `gemini-3-flash`）以及 API 请求的 Base URL。
4.  **工具类**：`scripts/gemini-webapi/utils/`
    *   **看点**：重点看 Cookie 的提取逻辑，了解它是如何从 Chrome 目录读取信息的。

## 3. 核心学习任务

### 第一阶段：黑盒分析 (运行与观察)
*   [ ] 成功运行 `npx -y bun main.ts --login`。
*   [ ] 观察生成一份图片请求时，控制台的网络活动（如果加了日志）。
*   [ ] 检查生成的 `consent.json` 和 `sessions/` 目录下的缓存文件。

### 第二阶段：逻辑追踪 (源码解析)
*   [ ] **任务 1**：找出程序是如何在没有 API Key 的情况下通过 `__Secure-1PSID` 进行授权的。
*   [ ] **任务 2**：在 `client.ts` 中找到发送 Prompt 的地方，观察那个复杂的嵌套数组结构是什么样的。
*   [ ] **任务 3**：理解 `sessionId` 的作用，找到代码中保存 `conversationId` 和 `responseId` 的位置。

### 第三阶段：动手实践 (功能修改)
*   [ ] **练习 1**：修改 `main.ts`，让程序在生成图片后自动在终端输出图片的存放路径。
*   [ ] **练习 2**：在错误处理逻辑中添加一个新的自定义异常处理（比如网络代理失败时的友好提示）。
*   [ ] **练习 3**：尝试添加一个新参数 `--save-log`，将对话内容保存到本地文本文件。

---

## 4. 推荐进阶方向

1.  **现代 TS 开发标准**：学习项目中如何定义 `Namespace`、`Interface` 以及如何管理异步 Promise 链。
2.  **网络安全与反爬**：研究项目如何模拟合理的 User-Agent 和 Header 来规避 Google 的风险检测。
3.  **跨平台脚本编写**：查看代码中是如何根据操作系统（Windows/macOS/Linux）动态调整路径的。

---

> [!TIP]
> **学习建议**：
> 不要试图一眼看穿所有的嵌套数组逻辑（那部分是非常枯燥的序列化工作），应该重点关注 **数据是如何流动的**：命令行参数 -> Client 处理 -> 发送 Fetch 请求 -> 处理返回的 Stream -> 输出结果。
